# Security Policy

### Supported Versions

We currently provide security updates and bug fixes for the following version of `osupp`:

| Version | Supported          |
| ------- | ------------------ |
| 0.5.x   | ✅ Yes             |
| < 0.5.0 | ❌ No (development only) |

> **Note:** Version `0.5.0` is the first stable release. Any earlier versions (0.4.x or below) are considered experimental and are not maintained.

### Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability within `osupp`, please follow these steps:

1. **Do not** open a public GitHub issue.
2. **Do not** disclose the vulnerability publicly until it has been addressed.
3. Send an email to **security@olib.dev** with the subject line:  
   `[osupp] Security Vulnerability Report`
4. Include as much information as possible:
   - Affected version(s)
   - Steps to reproduce
   - Potential impact
   - Any suggested fix (if available)

You can expect the following response timeline:

| Step | Timeframe |
|------|------------|
| Initial acknowledgment | Within 48 hours |
| Vulnerability assessment | Within 5 business days |
| Fix released (if accepted) | Within 14 business days (depending on severity) |

### What to Expect

- **Confirmed vulnerability:** We will work on a patch, credit you in the release notes (unless you prefer to remain anonymous), and publish a security advisory on GitHub.
- **Declined report:** If the issue is not considered a security vulnerability (e.g., configuration issue, non-exploitable behavior), we will explain why and may still address it as a regular bug fix.

### Preferred Languages

We prefer security reports to be submitted in **English**.

### Disclosure Policy

We follow responsible disclosure:

- A private report is received
- The issue is confirmed and a fix is prepared
- A patch is released privately or via a normal update
- After the fix is publicly available, a security advisory is published

### Thank You

We appreciate your efforts to keep `osupp` and its users safe. Contributors who report valid security issues will be mentioned in the `README.md` credits (unless they request otherwise).
